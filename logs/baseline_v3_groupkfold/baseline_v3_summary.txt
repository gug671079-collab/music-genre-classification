音乐风格自动分类系统 - Baseline V3 GroupKFold
============================================================

一、实验目的
实现一个更严格、更公平的未调参 KNN baseline。
该 baseline 使用 track_id 分组的 GroupKFold，用于后续调参模型的对照。

二、数据路径
读取数据：data/processed/processed_features_3_sec_V2.csv

三、数据规模
样本数量：9857
track_id 数量：987

四、类别分布
label
blues        1000
classical     998
country       997
disco         989
hiphop        978
jazz         1000
metal         930
pop           980
reggae        987
rock          998

五、字段处理
目标变量：label
分组字段：track_id
删除的非模型字段：
filename, genre_from_filename, track_id, segment_id, length

六、模型设置
使用的模型：KNN
KNN 固定参数：{'n_neighbors': 5, 'weights': 'uniform', 'metric': 'minkowski', 'p': 2}
说明：本 baseline 不进行调参，不使用 GridSearchCV，不搜索最优参数。

七、验证方式
验证方式：GroupKFold(n_splits=5)
分组字段：track_id

八、Baseline V3 与 Baseline V2 的区别
Baseline V3 与 Baseline V2 的主要区别在于评估方式。Baseline V2 使用片段级 train_test_split，可能存在同一首音乐的不同片段跨训练集和测试集的问题，因此结果可能偏乐观。Baseline V3 使用 GroupKFold，并以 track_id 为分组单位，能够保证同一首音乐的所有片段不会同时出现在训练折和验证折中，因此更适合作为后续调参实验的基准结果。
